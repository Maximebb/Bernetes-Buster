# Security Policy

## Supported versions

Only the latest `main` tip is supported.

## Reporting a vulnerability

Please **do not** open a public issue for security flaws in Bernetes-Buster itself.

Email or privately message the maintainer (@Maximebb on GitHub) with:

- Description and impact
- Reproduction steps
- Affected commit / tag

We aim to acknowledge reports within 7 days.

## Cluster assessments

If Bernetes-Buster discovers real issues in *your* cluster, treat the JSON report as sensitive and rotate any exposed credentials.
